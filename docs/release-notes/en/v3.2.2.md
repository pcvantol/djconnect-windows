# DJConnect Windows 3.2.2

- Aligns Ask DJ and Track Insight with the Home Assistant DJConnect `3.2.2`
  contract.
- Uses Music DNA terminology throughout the client and removes legacy memory
  wording and payload examples.
- Adds direct Track Insight from Now Playing and renders normalized
  `track_insight` results, including Music DNA Match, vibe hints and
  no-track-playing empty states.
- Routes Ask DJ Track Insight intents into the shared Track Insight surface
  without synthesizing playback buttons or stale media.
- Adds an optional local Home Assistant `/api/websocket` fast path for
  supported commands, Ask DJ messages and Track Insight, with HTTP kept as the
  canonical fallback.
- Adds the Windows Track Insight platform roadmap for dedicated navigation,
  demo provider, auto Track Insight and future visualizer work.
- Tightens Windows app pairing to the outbound Home Assistant flow: local Home
  Assistant URL, Home Assistant pairing code and no Windows client address,
  app-generated code or inbound waiting state.
- Replaces the first-run welcome screen with a five-step DJConnect tour for Now
  Playing, Ask DJ, Track Insight, Queue and Mini-games.
